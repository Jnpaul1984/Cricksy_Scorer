## 6. Module 1 — Cricket System Fundamentals

Status: In Progress
Estimated Duration: 1 week
Purpose: Build the learner's foundation for understanding cricket formats, innings, overs, phases, player roles, and analyst observation habits.

### Learning Objectives

By the end of this module, the learner should be able to:

- explain the differences between T20, ODI, and Test cricket
- explain innings and overs
- identify match phases
- describe basic player roles
- recognize simple tactical shifts
- explain why match context matters to analysts
- identify beginner-level AI opportunities in cricket analysis
- connect cricket structure to future Cricksy features

## 7. Daily Class Session Format

Each personal class session should follow this structure unless the curriculum file is intentionally updated.

1. Session title
2. Curriculum location
3. Learning objectives
4. Theory
5. Real-world example
6. Analyst workflow
7. AI-assisted analysis connection
8. Cricksy feature connection
9. Exercise or discussion task
10. Assessment question or practical task
11. Grade or feedback
12. Notion/source-file update recommendation

---

## 8. Grading Categories

Each assessed session may be graded using these categories:

| Category | Meaning |
|---|---|
| Tactical Understanding | Understanding of cricket decisions and strategy |
| Observation Quality | Ability to notice important match details |
| Analyst Reasoning | Ability to explain why something matters |
| Communication | Ability to explain findings clearly |
| AI Systems Thinking | Ability to identify where AI can support analysis |
| Cricksy Integration Thinking | Ability to connect learning to future platform features |

Suggested scale:

- 90 to 100: Excellent
- 80 to 89: Strong
- 70 to 79: Competent
- 60 to 69: Needs improvement
- Below 60: Redo required

---

## 9. Anti-Drift Rules

- Do not start a new module until the current module completion gates are reviewed.
- Do not add advanced AI topics into beginner sessions unless they are clearly marked as preview only.
- Do not skip assessments.
- Do not mark Notion items complete unless the session evidence exists.
- If a lesson expands too widely, capture extra ideas in Notion as future module notes instead of changing the current lesson scope.
- Curriculum changes must be intentional and reflected in this file.
